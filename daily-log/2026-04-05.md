\## What I Learned

\- How to create a GitHub repo from the browser and clone it locally

\- `git --version` to check if Git is installed

\- `mkdir` creates folders, `ren` renames them

\- Git doesn't track empty folders — you need a `.gitkeep` placeholder file

\- `git add .` stages everything (the dot means "current folder, everything")

\- `git status` is my new best friend — shows me what's staged, modified, or untracked

\- LF vs CRLF line ending warnings are harmless on Windows

\- Markdown (.md) files render beautifully on GitHub, PDFs don't
